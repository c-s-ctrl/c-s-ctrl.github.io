将本地的网站部署到githubpage上

February 17, 2025 阅读时间：1 分钟
冰冻水的记忆
零碎指南
Hugo • 记录 • Github

阅读这篇内容的读者需要先配置好一个本地可运行的博客。如果没有请查看我的文章：建立网站指南 注意，下面提到的所有<username>都请改成你的github用户名，且无需<>,

Github上的仓库设置 链接到标题

首先需要创建一个以<username>.github.io命名的仓库，并设置为公有。(这里网上部分教程说再设置一个私有仓库储存md文件，然而笔者发现hugo生成的public文件本身就是html文件，无md文件上传，因此应该无需担心源码泄漏)

检查baseURL配置 链接到标题

需要将hugo.toml第一行改为

baseURL = "https://<username>.github.io/"

然后在命令行运行

# 删除 public 文件夹并重新生成网站
#(如果你原来没有public文件夹可无需删除)
rm -rf public/
hugo
# 推送更新到 GitHub

git remote add origin https://github.com/<username>/<username>.github.io.git
git add .
#这里相当于一个给github的注释，可自己修改
git commit -m "deploy" 
#如果你推送分支不叫main，那么这里需要改名
git push --force origin master:main

如果在git remote add ···报错error: remote origin already exists. 说明你之前已经添加过远程仓库了，

可以运行 git remote -v验证这个说法

需要先删除再添加，删除：

git remote rm origin

再git remote add ···，然后再add.、commit、push，耐心等待上传成功，然后再github仓库中

Setting->(Code and automation)Pages

在Build and deployment处选择deploy from a branch，其中选择你的main。

之后等待片刻就可以在GitHub Pages处看到生成好的网站。
